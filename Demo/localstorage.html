What is Cookies?

Before discussing Web Storage, we need to understand the mechanism it was designed to improve: HTTP Cookies. Cookies were introduced in 1994 by Lou Montulli at Netscape Communications. They were created to solve a fundamental problem with HTTP: statelessness. Each HTTP request is independent, with no memory of previous interactions. Cookies provided a mechanism for servers to store small pieces of data on the client.

Technical Limitations of Cookies

Limitation Description Impact
Size Constraint 4KB maximum per cookie Insufficient for complex application data
Network Overhead Sent with every HTTP request Increased bandwidth consumption
API Design String-based manipulation only Complex parsing required for read operations
Security Exposure Vulnerable to XSS and CSRF Additional security measures required
Scope Control Path and domain based Limited granularity

What is localStorage?

The localStorage API is part of the Web Storage API introduced in HTML5. It allows web applications to store data locally in the user's browser with no expiration time.

📅 Introduction

Introduced in 2009 as part of HTML5 specification by the W3C and WHATWG

💾 Storage Capacity

Typically 5-10MB per origin (domain)

🔄 Persistence

Data persists even after browser/tab closes

🔐 Security

Same-origin policy, accessible only to the same domain

Code Examples

// Set an item

localStorage.setItem('username', 'JohnDoe');

// Get an item

const user = localStorage.getItem('username');

// Remove an item

localStorage.removeItem('username');

// Clear all items

localStorage.clear();

// Storing objects (must stringify)

const user = { name: 'John', age: 30 };
localStorage.setItem('user', JSON.stringify(user));
const retrieved = JSON.parse(localStorage.getItem('user'));
